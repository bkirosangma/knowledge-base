> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bundle three deferred polish items from the M2 editor: bend/slide keyboard cycling (#15), dirty-state cross-file reset (#17), and a voice-1 render verification probe (#18). Also fold the post-merge handoff update for TAB-010 (PR #113) into this branch per the project rule that handoff edits ride alongside feature PRs.

**Architecture:** Op-shape extension on `add-technique` (`amount?` + `direction?`) parameterizes the existing bend/slide mutators without renaming any existing `Technique` union members; the keyboard hook reads note state from the score on each B/S press and dispatches the next cycle op stateless. `useTabContent` gains a `[path]`-keyed reset effect. `inverseOf` + `useTabEditHistory` extend pre-state capture so undo restores the precise cycle position (½ → undo → no-bend, full → undo → ½, etc.). #18 ships as a probe doc with the manual-observation step recorded as a deferred PR-time check.

**Tech Stack:** TypeScript, React, Vitest + RTL, alphaTab `@coderline/alphatab` v1.6.x. Spec: `docs/superpowers/specs/2026-05-04-tab-008b-design.md`.

**Pre-flight facts (verified before plan-write):**
- `SlideOutType.OutDown = 4` is the alphaTab enum member for slide-down (`alphaTab.d.ts:15331`). Existing engine code captures `Shift = 1` for slide-up; we add `OutDown` for slide-down.
- `buildTechniqueMutators` lives at `infrastructure/alphaTabEngine.ts:253`. Mutator signature is `(note) => void`. We change to `(note, op?) => void`; bend & slide read `op?.amount` / `op?.direction`; other techniques ignore the second arg.
- `inverseOf` lives at `features/tab/editHistory/inverseOf.ts:33`. Today its `add-technique` arm returns a no-preState `remove-technique`. We extend the `PreState` union with a `{ bendType?, bendPoints?, slideOutType? }` variant and route bend/slide ops through it.
- `useTabContent` (`features/tab/hooks/useTabContent.ts`) currently has no `[path]`-keyed reset effect; the existing `useEffect(() => void load(path), [load, path])` runs the load but does not reset `dirty` / `saveError` / pending timer.

```ts
import { findNote } from "../scoreNavigation";

interface NoteBendShape {
  bendType?: number;
  bendPoints?: { value: number }[] | null;
  slideOutType?: number;
}

const BEND_TYPE_BEND = 1;       // alphaTab BendType.Bend
const SLIDE_TYPE_SHIFT = 1;     // alphaTab SlideOutType.Shift (up)
const SLIDE_TYPE_OUTDOWN = 4;   // alphaTab SlideOutType.OutDown (down)

function readBendValue(note: NoteBendShape | null): 50 | 100 | null {
  if (!note || note.bendType !== BEND_TYPE_BEND || !note.bendPoints) return null;
  // Final bend point's value identifies the cycle position.
  const last = note.bendPoints[note.bendPoints.length - 1];
  if (!last) return null;
  if (last.value === 50) return 50;
  if (last.value === 100) return 100;
  return null;  // microbend or other; treat as "no recognised cycle state"
}

function readSlideDirection(note: NoteBendShape | null): "up" | "down" | null {
  if (!note || !note.slideOutType) return null;
  if (note.slideOutType === SLIDE_TYPE_SHIFT) return "up";
  if (note.slideOutType === SLIDE_TYPE_OUTDOWN) return "down";
  return null;
}
```
